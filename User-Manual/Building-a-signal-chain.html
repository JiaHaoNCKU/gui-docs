Building a signal chain¶

The Open Ephys GUI provides a great deal of flexibility by allowing the user to mix and match different plugins. However, this freedom can be daunting when you’re just starting out. This section will guide you through the steps required to build a standard signal chain for extracellular electrophysiology, which allows you to acquire, record, and visualize continuous signals and spikes. It assumes you’re already familiar with the general layout of the user interface, so we recommend reading through the previous section if you haven’t already.

Adding a data source¶

  1. Drag the File Reader from the Processor List, and drop it on the Signal Chain. In an actual experiment, this would be substituted with the data source of your choice, such as the Rhythm FPGA or Neuropixels PXI.

  2. Press the buttoned labeled F to select a file to read in. The File Reader accepts .oebin files, which contain information about one or more continuous data files in Binary format. If you downloaded the GUI as a .zip file, the example files are available in the DataFiles folder. If you downloaded the GUI’s source code, the example files can be found in Resources/DataFiles.

  3. Select the subprocessor to stream data from (one of several available continuous data files indexed by the .oebin file that was loaded). In this example, we’ll be using data_stream_16ch_cortex, which contains data from 8 stereotrodes implanted in mouse somatosensory cortex, sampled at 40 kHz.

Adding a Record Node¶

  1. Scroll to the bottom of the Processor List, then drag and drop a Record Node onto the Signal Chain. This will record all data that comes out of the plugin directly to the left of it. The orange bar on the left indicates the fraction of disk space available in your default data directory. To change the location where data will be saved, click the ellipsis (...) button in the upper right of the Record Node.

Visualizing continuous signals¶

  1. Drag and drop a LFP Viewer to the right of the Record Node.

  2. Click the “tab” button in the upper right of the LFP Viewer, to open the visualizer in the Viewport.

Acquiring data¶

  1. Click the “play” button to begin data acquisition. Once you’ve confirmed that the data appears in the LFP Viewer visualization, click the “play” button again to stop acquisition.

Detecting spikes¶

  1. Drag and drop a Bandpass Filter to the right of the LFP Viewer. With the default settings, this will filter data in a range that’s appropriate for detecting action potential waveforms.

Note

Prior to GUI version 0.5.0, it was not possible to continue a signal chain after a “Sink” processor such as the LFP Viewer. This restriction has now been lifted, in order to simplify the creation of complex signal chains. Data will pass through Sinks the same way it passes through Filters, except it will not be modified in any way.

  1. Drag and drop a Spike Sorter to the right of the Bandpass Filter. The Spike Sorter’s functionality is a superset of the Spike Detector.

  2. Use the arrow buttons in the upper left of the Spike Sorter to increase the number of electrodes to 8. Then, press the + button to add 8 stereotrodes. This will automatically create 8 pairs of channels that capture spike waveforms whenever the signal on one of the two channels crosses a threshold. It’s also possible to create single electrodes (1 linked channel) or tetrodes (4 linked channels).

Visualizing spikes¶

  1. Double-click on the names of the File Reader, Record Node, and LFP Viewer to collapse their plugin editors. Alternatively, you can use the arrow buttons on the far right side of the Signal Chain to browse through active plugin editors if some of them have disappeared.

  2. Drag and drop another Record Node to the right of the Spike Sorter. This will be used to save the spikes coming out of this plugin, which are not accessible to the first Record Node, since they are generated by a downstream plugin.

  3. Click the vertical bars on the left of the new Record Node to expose the subprocessor configuration interface. This Record Node only has one subprocessor (a set of synchronously sampled continuous data channels). Clicking on the vertical bar labeled SPO will show 16 channels highlighted in red. Click the “NONE” button to deselect these channels, since they are already being recorded by the original Record Node. Next, click the red button labeled “RECORD EVENTS” to disable event saving. Importantly, though, the “RECORD SPIKES” button should still be red, to indicate that any incoming spikes will be saved by this Record Node. Lastly, if you changed the data directory for the previous Record Node, you should do so again.

  4. Drag and drop a Spike Viewer to the right of the Record Node.

  5. Click the tab button in the upper right of the Spike Viewer to open the visualizer in the Viewport.

Listening to and recording spikes¶

  1. Click the “play” button to re-start acquisition. You should now see spikes being displayed by the Spike Viewer.

  2. While acquisition remains active, press the “MONITOR” button in the Spike Sorter to listen to the spikes from the selected electrode. If you can’t hear anything, check that the volume slider in the Control Panel is set to a value other than zero and that your computer’s speakers are not muted.
